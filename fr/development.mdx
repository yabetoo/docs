<Info>
  **Prérequis** : Veuillez installer Node.js (version 19 ou supérieure) avant de continuer. <br />
  Veuillez passer à ```docs.json``` avant de continuer et supprimer l'ancien fichier ```mint.json```.
</Info>

Suivez ces étapes pour installer et exécuter Mintlify sur votre système d'exploitation :

**Étape 1** : Installer Mintlify :

<CodeGroup>

  ```bash npm
  npm i -g mintlify
  ```

```bash yarn
yarn global add mintlify
```

</CodeGroup>

**Étape 2** : Naviguez vers le répertoire de documentation (où se trouve le fichier `docs.json`) et exécutez la commande suivante :

```bash
mintlify dev
```

Une prévisualisation locale de votre documentation sera disponible à l'adresse `http://localhost:3000`.

### Ports personnalisés

Par défaut, Mintlify utilise le port 3000. Vous pouvez personnaliser le port sur lequel Mintlify s'exécute en utilisant l'option `--port`. Pour exécuter Mintlify sur le port 3333, par exemple, utilisez cette commande :

```bash
mintlify dev --port 3333
```

Si vous essayez d'exécuter Mintlify sur un port déjà utilisé, il utilisera le prochain port disponible :

```md
Le port 3000 est déjà utilisé. Essai du port 3001 à la place.
```

## Versions de Mintlify

Veuillez noter que chaque version du CLI est associée à une version spécifique de Mintlify. Si votre site local n'est pas aligné avec la version de production, veuillez mettre à jour le CLI :

<CodeGroup>

```bash npm
npm i -g mintlify@latest
```

```bash yarn
yarn global upgrade mintlify
```

</CodeGroup>

## Validation des liens

Le CLI peut vous aider à valider les liens de référence dans votre documentation. Pour identifier les liens cassés, utilisez la commande suivante :

```bash
mintlify broken-links
```

## Formatage du code

Nous suggérons d'utiliser des extensions sur votre IDE pour reconnaître et formater MDX. Si vous utilisez VSCode, considérez l'[extension MDX VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) pour la coloration syntaxique, et [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) pour le formatage du code.

## Dépannage

<AccordionGroup>
  <Accordion title='Erreur : Impossible de charger le module "sharp" en utilisant le runtime darwin-arm64'>

    Cela peut être dû à une version obsolète de node. Essayez ce qui suit :
    1. Supprimez la version actuellement installée de mintlify : `npm remove -g mintlify`
    2. Passez à Node v19 ou supérieur.
    3. Réinstallez mintlify : `npm install -g mintlify`
  </Accordion>

  <Accordion title="Problème : Rencontre d'une erreur inconnue">
  
    Solution : Allez à la racine de votre appareil et supprimez le dossier \~/.mintlify. Ensuite, exécutez à nouveau `mintlify dev`.
  </Accordion>
</AccordionGroup>

Curieux de connaître les changements dans la version du CLI ? [Consultez le changelog du CLI.](https://www.npmjs.com/package/mintlify?activeTab=versions)
