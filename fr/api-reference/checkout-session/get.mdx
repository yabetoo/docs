---
title: Récupérer une session
description: Récupérez les détails d'une session de paiement existante.
---

## Endpoint

```bash
GET /v1/checkout/sessions/:id
```

## Paramètres

### Paramètres de chemin

| Paramètre | Description |
|-----------|-------------|
| `id` | **Obligatoire**. ID de la session de paiement |

## Exemple de requête

<CodeGroup>
```bash cURL
curl https://pay.sandbox.yabetoopay.com/v1/checkout/sessions/cs_test_123456789 \
  -H "Authorization: Bearer YOUR_SECRET_KEY"
```

```javascript JavaScript
const Yabetoo = require("yabetoo-sdk");
const yabetoo = new Yabetoo("YOUR_SECRET_KEY");

const session = await yabetoo.checkoutSessions.retrieve("cs_test_123456789");
```

```python Python
import yabetoo

yabetoo.api_key = "YOUR_SECRET_KEY"

session = yabetoo.CheckoutSession.retrieve("cs_test_123456789")
```

```php PHP
<?php
$yabetoo = new \Yabetoo\YabetooClient("YOUR_SECRET_KEY");

$session = $yabetoo->checkoutSessions->retrieve("cs_test_123456789");
```

```java Java
import com.yabetoo.Yabetoo;
import com.yabetoo.model.CheckoutSession;

Yabetoo.apiKey = "YOUR_SECRET_KEY";

CheckoutSession session = CheckoutSession.retrieve("cs_test_123456789");
```
</CodeGroup>

### Statuts possibles

| Statut | Description |
|--------|-------------|
| `pending` | Session créée, en attente de paiement |
| `complete` | Paiement effectué avec succès |
| `expired` | Session expirée sans paiement |
| `canceled` | Session annulée |

## Codes d'erreur

| Code | Description |
|------|-------------|
| `session_not_found` | Session introuvable |
| `invalid_session_id` | Format d'ID de session invalide |

<Note>
  Les sessions complétées ou expirées restent accessibles pendant 30 jours.
</Note>
